# pkg/llm Phase 0.5 — Codec Relocation (`openaiapi` → `codec/openaiapi` + `codec/sse`) — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task (fresh subagent per task, spec + code-quality review between tasks).

**Goal:** Relocate the OpenAI codec from `pkg/llm/openaiapi` to `pkg/llm/codec/openaiapi`, and hoist the shared SSE framing out of the codec into a new `pkg/llm/codec/sse` package — a mechanical, behavior-preserving move that establishes the codec axis from the layout design, with **zero change to any codec logic**.

**Architecture:** This is a *relocation refactor*, not a feature. The safety net is the **existing test suite**: every test moves with its package and must stay green after the move — that is the regression proof that behavior is unchanged. There is no "write a failing test first" here; the discipline is "the moved tests keep passing and the whole module still builds." Package *names* stay the same (`openaiapi`); only **import paths** change. The one small idiomatic improvement is renaming `openaiapi.SSEReader`/`NewSSEReader` → `sse.Reader`/`sse.NewReader` when it lands in its own package (avoids the `sse.SSEReader` stutter that staticcheck ST1003 flags).

**Tech Stack:** Go (module `github.com/looprig/harness`), stdlib only. `git mv` to preserve history. Tests under `-race`. Design source: `docs/plans/2026-07-01-llm-provider-codec-layout-design.md` (§ Package layout, § Migration → Phase 0.5).

**Blast radius (verified 2026-07-02):** The codec's only importers are:
- `pkg/llm/aci/client.go` — imports `openaiapi` directly; uses `EncodeRequest` (×2), `DecodeResponse` (×1), `NewSSEReader` (×1, line ~428).
- `pkg/llm/openaiapi/chutes/*` (decode.go, encode.go, stream.go, client_test.go) — import `openaiapi`; use `BuildChatRequest`, `ChatRequest`, `DecodeResponse`, `NewStream`. **chutes has its OWN encrypted SSE reader (`ssereader.go`)** — it does NOT use the hoisted `SSEReader`, so the SSE hoist does not touch chutes.
- `pkg/llm/openaiapi/lmstudio/*` (client.go, decode.go, encode.go, client_test.go) — import `openaiapi`; use `DecodeResponse`, `EncodeRequest`, `NewStream`.
- `pkg/llm/auto/auto.go` — imports the **subpackages** `openaiapi/chutes` and `openaiapi/lmstudio`, **not** the codec directly.
- The sibling `swe` consumer module does **not** import the codec directly (uses `auto`/`llm` public API). Out of scope.

**Scope guardrails:**
- **Do NOT move `chutes`/`lmstudio` directories** in this phase. They stay at `pkg/llm/openaiapi/chutes` and `pkg/llm/openaiapi/lmstudio`; only their `openaiapi` *import path* is updated to `codec/openaiapi`. (They dissolve into `providers/` in Phase 1 — moving them now is churn.) This leaves `pkg/llm/openaiapi/` as a parent dir holding only those two subpackages, which is fine and temporary.
- **Do NOT change any codec logic, signature, or behavior.** The only symbol rename is `SSEReader`/`NewSSEReader` → `Reader`/`NewReader` (and only because it moves into package `sse`). Every other exported name (`BuildChatRequest`, `ChatRequest`, `EncodeRequest`, `DecodeResponse`, `NewStream`) keeps its name.
- **Do NOT add `codec/anthropicapi`** — that lands only when a real Anthropic-format provider is needed (design § Migration; YAGNI).
- **Do NOT touch the design doc** or Phase 0 symbols.

**Worktree / go.work note (CRITICAL for every command):** Implementation happens in a git worktree under `.worktrees/`, which lives **outside** the parent `/Users/ipotter/code/go.work`. Therefore **every `go`/`make` command MUST be prefixed with `GOWORK=off`** (e.g. `GOWORK=off go test -race ./...`, `GOWORK=off make secure`). Without it: `directory prefix ... does not contain modules listed in go.work`. Never edit `go.work`.

### Task 2: Hoist SSE framing → new package `pkg/llm/codec/sse`

**Rationale:** The layout design puts SSE framing in a shared `codec/sse` so both dialects (OpenAI now, Anthropic later) reuse one de-framer. Extract it now so the codec package owns *semantics*, not *framing* (matches the `Codec` interface doc-comment added in Phase 0).

**Current symbols (in `pkg/llm/codec/openaiapi/sse.go` after Task 1):** `type SSEReader struct{...}`, `func NewSSEReader(r io.Reader) *SSEReader`, method `(*SSEReader) Next() (...)`. Consumers: `codec/openaiapi/stream.go` (`NewSSEReader(body)` at ~line 15) and `pkg/llm/aci/client.go` (`openaiapi.NewSSEReader(...)` at ~line 428).

**Files:**
- Create: `pkg/llm/codec/sse/sse.go` (new `package sse`)
- Create/move test: `pkg/llm/codec/sse/sse_test.go` (from `codec/openaiapi/sse_test.go`)
- Delete: `pkg/llm/codec/openaiapi/sse.go` (contents move to the new package)
- Modify: `pkg/llm/codec/openaiapi/stream.go` (use `sse.NewReader`)
- Modify: `pkg/llm/aci/client.go` (use `sse.NewReader`; add `codec/sse` import)

**Step 1 — Create `pkg/llm/codec/sse/sse.go`.** Move the SSE reader verbatim into `package sse`, renaming to drop the stutter:
