# CLI TUI Design

## Scope

This spec covers the Nexus CLI тАФ a terminal UI that connects a human to an
`internal/agent/session.Session`. It is structured in two layers:

- `client/` тАФ platform-neutral session client (reusable by future Wails desktop app)
- `client/cli/tui/` тАФ Bubbletea TUI that wraps the client for terminal rendering
- `cmd/cli/main.go` тАФ binary entrypoint, wiring only

**What is built here:** core chat loop only тАФ send turns, stream token deltas, render
finalized responses. No tools, no permission prompts, no session picker yet. Those are
added as the agent loop grows.

## client/message.go

### DisplayRole

```go
type DisplayRole uint8

const (
    RoleUser        DisplayRole = iota
    RoleAssistant
    RoleSystem
    RoleError
    RoleInterrupted  // tombstone тАФ no content; renders as "тФФтФА interrupted"
)
```

Typed constant, not a string. `RoleError` and `RoleInterrupted` are TUI-only concepts
that have no equivalent in `llm.AgenticRoleType`.

### DisplayMessage

```go
type DisplayMessage struct {
    Role    DisplayRole
    Content []*llm.ContentBlock  // all roles; nil for RoleInterrupted
}
```

`Content` is `[]*llm.ContentBlock` for all roles. System and error messages are wrapped
in a single text block on creation. This keeps the renderer uniform тАФ it always
iterates `Content` and dispatches on `ContentBlockType`, with no special-cased string
fields. When thinking blocks and tool calls arrive in future chunks they slot in without
touching this type.

### ClientStatus

```go
type ClientStatus uint8

const (
    ClientIdle         ClientStatus = iota // no turn running; Submit sends immediately
    ClientRunning                          // turn in flight; Submit queues
    ClientInterrupting                     // Interrupt sent; waiting for Interrupted event
)
```

Named to mirror `loopIdle` / `loopRunning` in `internal/agent/loop/loop.go`.
`ClientInterrupting` has no loop equivalent тАФ it guards against sending multiple
`Interrupt` commands before the first one is acknowledged.

## client/input.go

### Input

Builds `[]*content.Block` for `loop.StartTurn`. Uses `content.Block` and
`content.ImageBlock` from `internal/content` directly тАФ no new types defined.

```go
type Input struct {
    blocks []*content.Block
}

func NewInput(text string) *Input {
    return &Input{blocks: []*content.Block{
        {Type: content.TypeText, Text: text},
    }}
}

func (i *Input) AddText(text string) *Input {
    i.blocks = append(i.blocks, &content.Block{
        Type: content.TypeText, Text: text,
    })
    return i
}

func (i *Input) AddImage(mediaType string, data []byte) *Input {
    i.blocks = append(i.blocks, &content.Block{
        Type:  content.TypeImage,
        Image: &content.ImageBlock{MediaType: mediaType, Data: data},
    })
    return i
}

func (i *Input) AddImageURL(url string) *Input {
    i.blocks = append(i.blocks, &content.Block{
        Type:  content.TypeImage,
        Image: &content.ImageBlock{URL: url},
    })
    return i
}

func (i *Input) Blocks() []*content.Block { return i.blocks }
```

### AddFile

```go
var plaintextExts = map[string]bool{
    ".txt": true, ".md": true, ".go": true, ".py": true, ".js": true,
    ".ts": true, ".json": true, ".yaml": true, ".yml": true, ".toml": true,
    ".sh": true, ".env": true, ".csv": true, ".html": true, ".xml": true,
    ".rs": true, ".java": true, ".c": true, ".cpp": true, ".h": true,
}

func (i *Input) AddFile(path string) (*Input, error)
```

Reads a file from disk and appends it as a text block:

1. Check `filepath.Ext(path)` against `plaintextExts`. If not recognised, return
   `fmt.Errorf("unsupported file type %s тАФ only plaintext files are supported", ext)`.
2. Read file contents via `os.ReadFile`.
3. Append `content.Block{Type: content.TypeText, Text: "[" + filepath.Base(path) + "]\n" + string(data)}`.

PDFs are explicitly not supported тАФ models do not accept raw PDF bytes and there is no
pure-Go PDF text extraction worth pulling in as a dependency. Users needing PDF content
should extract text externally before attaching.

`ThinkingBlock` is intentionally absent тАФ thinking is generated by the model, never
submitted by the user.

## client/client.go

### Client

Platform-neutral. Owns all phase logic, input queue, display history, and the event
drain goroutine. Never imports Bubbletea, Wails, or any rendering package.

```go
type Client struct {
    commands chan<- loop.Command

    mu      sync.RWMutex
    status  ClientStatus
    history []DisplayMessage
    stream  string    // accumulates TokenDelta.Text; cleared on TurnDone/TurnFailed/Interrupted
    queue   []*Input  // FIFO; popped one at a time as turns complete
}

func New(commands chan<- loop.Command) *Client
```

`mu` is a `sync.RWMutex`. `Listen` and `Submit`/`Interrupt`/`Shutdown` acquire the
write lock (`mu.Lock`). `Read` acquires the read lock (`mu.RLock`) тАФ multiple renders
can proceed concurrently without blocking each other or the write path.

### Listen

```go
func (c *Client) Listen(events <-chan loop.Event, notify func())
```

Spawns a goroutine that drains `events` until the channel closes:

```go
go func() {
    for ev := range events {
        c.mu.Lock()
        c.handleEvent(ev)
        c.mu.Unlock()
        notify()
    }
}()
```

`notify` is a platform callback called after every state change:
- **TUI:** `func() { prog.Send(refreshSignal{}) }` тАФ triggers a Bubbletea re-render
- **Wails (future):** `func() { runtime.EventsEmit(ctx, "state", ...) }`

`main.go` calls `Listen` once after `session.New`. It does not manage a drain goroutine
itself тАФ `Listen` owns it.

### handleEvent (unexported)

Called under `mu`. Mutates client state.

| Event | Action |
|---|---|
| `loop.SessionStarted` | append `DisplayMessage{RoleSystem, textBlocks("session ready")}` |
| `loop.TurnStarted` | `status = ClientRunning`; clear `stream` |
| `loop.TokenDelta` | `stream += ev.Text` |
| `loop.TurnDone` | append `DisplayMessage{RoleAssistant, ev.Content}`; clear `stream`; `status = ClientIdle`; pop queue |
| `loop.TurnFailed` | append `DisplayMessage{RoleError, textBlocks(ev.Err)}`; clear `stream`; `status = ClientIdle`; pop queue |
| `loop.TurnRejected` | append `DisplayMessage{RoleError, textBlocks(ev.Reason)}`; `status = ClientIdle`; pop queue |

**Interrupted event (Chunk 4):** when `loop.Interrupted` arrives:
- flush `stream` into history as `RoleAssistant` if non-empty (partial response)
- append `DisplayMessage{RoleInterrupted, nil}`
- clear `stream`; `status = ClientIdle`; pop queue

**Popping the queue:**
```
if len(c.queue) > 0:
    next := c.queue[0]
    c.queue = c.queue[1:]
    send loop.StartTurn{Input: next.Blocks()} to c.commands
    status = ClientRunning  (new turn already in flight)
```

`textBlocks` is a package-level helper:

```go
func textBlocks(text string) []*llm.ContentBlock {
    return []*llm.ContentBlock{{
        Type: llm.ContentBlockTypeText,
        Text: &llm.TextBlock{Text: text},
    }}
}
```

For `TurnDone`, `ev.Content` (`[]*llm.ContentBlock`) is stored directly тАФ no
extraction. `stream` is used only for live rendering during streaming; `TurnDone`
carries the finalized content from the loop as the source of truth.

### Submit

```go
func (c *Client) Submit(input *Input)
```

Acquires `mu`.

- `ClientIdle`: appends `DisplayMessage{RoleUser, inputToBlocks(input)}` to history.
  Sends `loop.StartTurn{Input: input.Blocks()}` to `commands`.
  Sets `status = ClientRunning`.
- `ClientRunning`: appends `DisplayMessage{RoleUser, inputToBlocks(input)}` to history.
  Appends `input` to `c.queue`.
  The renderer derives queued status from `queueLen` in `Read(fn)`.
- `ClientInterrupting`: drops the input silently.

`inputToBlocks` converts `[]*content.Block` тЖТ `[]*llm.ContentBlock`. For each block:
- `content.TypeText` тЖТ `&llm.ContentBlock{Type: llm.ContentBlockTypeText, Text: &llm.TextBlock{Text: b.Text}}`
- `content.TypeImage` тЖТ `&llm.ContentBlock{Type: llm.ContentBlockTypeImage, Image: b.Image}` (reuses the `*content.ImageBlock` pointer directly тАФ `llm.ContentBlock.Image` is already `*content.ImageBlock`)

This is the same mapping `loop/turn.go`'s `userMessage()` applies.

### Interrupt

```go
func (c *Client) Interrupt()
```

Acquires `mu`. `ClientRunning` only: sends `loop.Interrupt{}` to `commands`, sets
`status = ClientInterrupting`. Any other status: no-op.

`loop.Interrupt` does not exist yet (Chunk 4). Until it lands this method is a
documented no-op.

### Shutdown

```go
func (c *Client) Shutdown()
```

Sends `loop.Shutdown{}` unconditionally. Called when the user exits.

### Read

```go
func (c *Client) Read(fn func(history []DisplayMessage, stream string, status ClientStatus, queueLen int))
```

`ClientStatus` is exported so `tui/components` can reference it without importing an unexported type across package boundaries.

Acquires the read lock for the duration of `fn`. Passes the live slice directly тАФ no
copy. Multiple `Read` calls can proceed concurrently. `fn` must not mutate the slice or
store references beyond the call тАФ the lock is released when `fn` returns.

`queueLen` tells the renderer how many trailing user messages in `history` are still
pending (queued). Platform renderers call this; they never read `Client` fields directly.

## client/slash.go

Slash commands live in `client/` so they are shared across all platform clients (TUI,
Wails). `Run` operates on `*Client` only тАФ no Bubbletea or Wails imports.

```go
type SlashCmd struct {
    Name string
    Desc string
    Run  func(*Client)
}

var SlashCmds []SlashCmd // populated in Chunk 4 once Clear/appendHelp exist
```

`SlashCmds` is an empty slice in this chunk тАФ the registry is wired but no commands
are registered until Chunk 4 defines `Clear` and `appendHelp` on `*Client`.

Platform-specific commands (e.g. `/quit` тЖТ `tea.Quit`) are not in this registry.
The TUI handles them locally after checking against `SlashCmds`.

## TUI architecture тАФ The Elm Architecture (TEA)

Bubbletea follows TEA тАФ the same model underlying React/Redux:

| React/Redux | Bubbletea |
|---|---|
| Component state | `Model` struct (value type) |
| Action | `tea.Msg` |
| Reducer | `Update(msg) (Model, tea.Cmd)` |
| Render | `View() string` |
| Side effect / `useEffect` | `tea.Cmd` returned from `Update` |

**Critical invariant:** `Screen` and all component structs are **value types**. `Update`
returns a new copy тАФ it does not mutate the receiver. All field writes happen on the
local copy before returning. Bubbletea owns the canonical model; callers never hold
a pointer to it.

**`tea.Cmd` batching:** every `Update` that delegates to sub-components collects their
returned `tea.Cmd` values and returns `tea.Batch(cmds...)`. Dropping a `cmd` silently
breaks blinking, ticking, or async operations.

## client/cli/tui/components/

Each component owns its state (struct), its reducer (`Update`), and its render
(`View`). `Screen` composes them; it never reaches into component internals directly.

### components/history.go тАФ ChatHistory

```go
type ChatHistory struct {
    scroll viewport.Model
    cache  map[int]string // history index тЖТ rendered markdown; invalidated on resize
    width  int
}

func NewChatHistory(width, height int) ChatHistory
func (h ChatHistory) Update(msg tea.Msg) (ChatHistory, tea.Cmd)
func (h ChatHistory) View() string
func (h ChatHistory) Refresh(c *client.Client) ChatHistory
func (h ChatHistory) Resize(width, height int) ChatHistory
func (h ChatHistory) Clear() ChatHistory
```

**State:** `viewport.Model` (scroll position, visible area), `cache` (memoised markdown
renders), `width` (needed by renderer for line wrapping).

**`Refresh(c)`:** called by `Screen.Update` on `refreshSignal`. Reads client state via
`c.Read(...)`, renders all messages into a string, calls `h.scroll.SetContent`. Returns
updated `ChatHistory`. Auto-scrolls to bottom only if the viewport was already at the
bottom before the update тАФ preserves user scroll position.

**`Resize(width, height)`:** sets `h.scroll` dimensions, invalidates `cache` (sets it
to a new empty map), re-renders content.

**`Clear()`:** resets `cache` to an empty map and calls `h.scroll.SetContent("")`.
Returns updated `ChatHistory`. Called by `Screen.Update` after `/clear` dispatches
to the client.

**`Update`:** delegates `msg` to `h.scroll.Update(msg)` to handle keyboard/mouse scroll
events. Returns updated `ChatHistory` and any `tea.Cmd` from the viewport.

### components/input.go тАФ InputBox

```go
type InputBox struct {
    ta textarea.Model
}

func NewInputBox(width int) InputBox
func (b InputBox) Update(msg tea.Msg) (InputBox, tea.Cmd)
func (b InputBox) View() string
func (b InputBox) Value() string
func (b InputBox) Reset() InputBox
func (b InputBox) SetValue(s string) InputBox
func (b InputBox) Resize(width int) InputBox
```

**State:** `textarea.Model` (cursor position, text content, focus state).

**Initialization (`NewInputBox`):**
- `textarea.New()` with `CharLimit(0)` (unlimited)
- `SetHeight(3)` тАФ fixed 3-line input area
- `Focus()` тАФ input starts focused
- `ShowLineNumbers(false)`

**`Update`:** delegates `msg` to `b.ta.Update(msg)`. Returns updated `InputBox` and
`tea.Cmd` (carries the blink tick).

**`Value()`:** returns `b.ta.Value()` тАФ current text content.

**`Reset()`:** calls `b.ta.Reset()`, returns updated `InputBox`.

**`SetValue(s)`:** calls `b.ta.SetValue(s)`, returns updated `InputBox`. Used by Tab completion to fill the accepted command name into the textarea.

**`Resize(width)`:** calls `b.ta.SetWidth(width - 4)` (subtracts border and padding already applied by `styles.InputStyle`), returns updated `InputBox`.

Returns `styles.StatusStyle.Render(text)` for the active status:

|---|---|

### components/slashcomplete.go тАФ SlashComplete

```go
type SlashComplete struct {
    cmds   []client.SlashCmd // filtered subset matching current prefix
    cursor int               // selected index; wraps within len(cmds)
}

func NewSlashComplete(prefix string) *SlashComplete // nil if no matches
func (s SlashComplete) Update(msg tea.Msg) (SlashComplete, tea.Cmd)
func (s SlashComplete) View(width int) string
func (s SlashComplete) Selected() client.SlashCmd
```

**State:** filtered command list + cursor position.

**`NewSlashComplete(prefix)`:** filters `client.SlashCmds` by prefix. Returns `nil` if
no matches тАФ nil pointer on `Screen` means panel hidden.

**`Update`:** handles `тЖС/тЖУ` to move cursor (wrapping). Returns updated `SlashComplete`
and nil `tea.Cmd` (no async ops).

**`View(width)`:** renders the panel using `styles.CompPanelStyle`, `styles.CompItemStyle`,
`styles.CompSelectedStyle`, `styles.CompDescStyle`. Selected item highlighted in navy `"33"`.

**`Selected()`:** returns `cmds[cursor]`.

## client/cli/tui/model.go

### Screen

```go
type Screen struct {
    client        *client.Client
    history       components.ChatHistory
    input         components.InputBox
    slashComplete *components.SlashComplete // nil = panel hidden
    width         int
    height        int
    ready         bool // false until first tea.WindowSizeMsg
}

func New(c *client.Client) Screen
```

`Screen` holds no phase, no message history, no stream тАФ those live in `*client.Client`.
It is a pure composition root.

**`New`:** constructs components with zero size тАФ they are resized on the first
`tea.WindowSizeMsg`. Concretely:
```go
func New(c *client.Client) Screen {
    return Screen{
        client:  c,
        history: components.NewChatHistory(0, 0),
        input:   components.NewInputBox(0),
    }
}
```
`ready` starts `false`; `View` returns `"initializingтАж"` until the first resize.

### signal

```go
type signal interface{ isSignal() }

type refreshSignal struct{}

func (refreshSignal) isSignal() {}
```

Marker interface for all TUI-internal messages sent via `prog.Send`. `refreshSignal` is
the only implementation today. Future signals add a new struct implementing `isSignal()`.

### Notify

```go
func Notify(prog *tea.Program) func() {
    return func() { prog.Send(refreshSignal{}) }
}
```

Returns the `notify` closure for `client.Listen`. Keeps `refreshSignal` and `signal`
unexported; `main.go` never touches internal TUI types.

### Init

```go
func (m Screen) Init() tea.Cmd { return textarea.Blink }
```

### Update

`Update` is the single reducer for the root model. It handles top-level messages first,
then delegates to sub-components, then batches all returned `tea.Cmd` values.

```go
func (m Screen) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    var cmds []tea.Cmd
    var cmd tea.Cmd

    switch msg := msg.(type) {
    case tea.WindowSizeMsg:
        // size all components; set ready = true
    case signal:
        switch msg.(type) {
        case refreshSignal:
            m.history = m.history.Refresh(m.client)
        }
    case tea.KeyMsg:
        // key dispatch (see table below)
        // NOTE: key handling returns early for submit/quit тАФ
        // component delegation still runs for all other keys
    }

    m.history, cmd = m.history.Update(msg); cmds = append(cmds, cmd)
    m.input, cmd   = m.input.Update(msg);   cmds = append(cmds, cmd)
    if m.slashComplete != nil {
        sc, cmd := m.slashComplete.Update(msg)
        m.slashComplete = &sc
        cmds = append(cmds, cmd)
    }

    return m, tea.Batch(cmds...)
}
```

**Key handling table:**

| Key | Condition | Action |
|---|---|---|
| `Enter` (no Alt) | slashComplete visible | call `Selected().Run(m.client)`; `m.input.Reset()`; nil slashComplete |
| `Enter` (no Alt) | input empty | no-op |
| `Enter` (no Alt) | starts with `/` | scan `client.SlashCmds` for match тЖТ `cmd.Run(m.client)`; if name is `/clear` also call `m.history.Clear()`; if `/quit` тЖТ `m.client.Shutdown()`, return `tea.Quit`; no match тЖТ submit as plain text |
| `Enter` (no Alt) | otherwise | `m.client.Submit(client.NewInput(m.input.Value()))`; `m.input.Reset()` |
| `Esc` | any | `m.client.Interrupt()`; nil slashComplete |
| `Ctrl+C` | any | `m.client.Shutdown()`; return `tea.Quit` |
| `Tab` | slashComplete visible | `m.input.SetValue(m.slashComplete.Selected().Name)`; nil slashComplete |
| any printable | input starts with `/`, no space | rebuild slashComplete via `NewSlashComplete(prefix)` |

### Update тАФ window resize

```
history height  = terminal height - inputBox height (3) - border (2) - status line (1)
inputBox width  = terminal width - border (2) - padding (2)
history width   = terminal width
```

On `tea.WindowSizeMsg`: call `m.history.Resize(...)`, `m.input.Resize(...)`, set
`m.width`, `m.height`, `m.ready = true`.

### View

```go
func (m Screen) View() string {
    if !m.ready {
        return "initializingтАж"
    }
    var statusLine string
    m.client.Read(func(_ []client.DisplayMessage, _ string, status client.ClientStatus, _ int) {
        statusLine = components.RenderStatusLine(status)
    })
    inputBox := styles.InputStyle.Width(m.width - 2).Render(m.input.View())
    if m.slashComplete != nil {
        return lipgloss.JoinVertical(lipgloss.Left,
            m.history.View(), statusLine, m.slashComplete.View(m.width), inputBox)
    }
    return lipgloss.JoinVertical(lipgloss.Left,
        m.history.View(), statusLine, inputBox)
}
```

## client/cli/tui/components/render.go

Render helpers shared across components. Imports `tui/styles` тАФ no cycle.

### renderMD

```go
func renderMD(content []*llm.ContentBlock, width int, cache map[int]string, idx int) string
```

Extracts text via `llm.TextContent(content)`, renders via glamour using `styles.MdStyle`.
Trims surrounding newlines. Prefixes first line with `styles.Dot`; indents continuation
lines by `styles.DotWidth` spaces. Falls back to `styles.Dot + " " + text` on glamour
error or `width == 0`. Result cached in `cache[idx]` on first render.

### renderMessages

```go
func renderMessages(history []client.DisplayMessage, stream string, queueLen int, width int, cache map[int]string) string
```

Iterates `history`. The last `queueLen` user messages are pending тАФ rendered with
`styles.UserQueuedBubbleStyle`. Dispatches on `Role`:

| Role | Render |
|---|---|
| `client.RoleUser` (not queued) | `styles.UserBubbleStyle.Width(width-4).Render(llm.TextContent(msg.Content))` |
| `client.RoleUser` (queued) | `styles.UserQueuedBubbleStyle.Width(width-4).Render(llm.TextContent(msg.Content) + " (queued)")` |
| `client.RoleAssistant` | `renderMD(msg.Content, width, cache, i)` |
| `client.RoleSystem` | `styles.SystemMsgStyle` dot + `llm.TextContent(msg.Content)` |
| `client.RoleError` | `styles.ErrorMsgStyle` dot + `llm.TextContent(msg.Content)` |
| `client.RoleInterrupted` | `"  тФФтФА " + styles.InterruptedStyle.Render("interrupted")` |

If `stream != ""`, appends a live assistant entry:
`styles.AssistantDotStyle.Render(styles.Dot) + " " + styles.AssistantMDStyle.Render(stream + "тАж")`

Separator: double newline between role changes; single newline between same-role
consecutive messages.

Called by `ChatHistory.Refresh` тАФ not by `Screen` directly.

### wordWrap / wrapText

```go
func wordWrap(text string, width int) []string
func wrapText(text string, width int, indent int, style lipgloss.Style) string
```

`wordWrap` splits `text` into lines at word boundaries so no line exceeds `width` тАФ
pure string logic, no styling. `wrapText` calls `wordWrap`, renders each line with
`style`, and indents continuation lines by `indent` spaces. Used by `renderMessages`
for system and error messages.

## cmd/cli/main.go

### Startup sequence

```
1. ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGTERM, syscall.SIGINT)
   defer stop()
2. Parse flags: --model, --system, --auth-flow
3. Read NEXUS_GATEWAY_URL; fail fast if empty and NEXUS_NO_AUTH != "true"
4. Auth:
   - NEXUS_NO_AUTH=true      тЖТ token closure returns ("", nil)
   - --auth-flow=devicecode  тЖТ auth.NewDeviceCodeFlow(...).Authenticate(ctx)
   - default                 тЖТ auth.NewAuthCodeFlow(...).Authenticate(ctx)
5. Build Token closure with auto-refresh (calls flow.Refresh on NeedsRefresh)
6. cfg := loop.Config{GatewayURL: ..., Token: token, Model: llm.ModelSpec{Model: modelFlag, System: systemFlag}}
7. sess, err := session.New(ctx, cfg)
8. c := client.New(sess.Commands)
9. screen := tui.New(c)
10. prog := tea.NewProgram(screen, tea.WithAltScreen())
11. c.Listen(sess.Events, tui.Notify(prog))
12. prog.Run()
```

SIGTERM/SIGINT cancels `ctx` тЖТ loop's `listen` goroutine handles `<-ctx.Done()` and
closes `sess.Events` тЖТ `Listen` goroutine exits тЖТ `prog.Run` returns.

Ctrl+C from keyboard: Bubbletea key handler calls `client.Shutdown()` тЖТ sends
`loop.Shutdown{}` тЖТ loop drains gracefully тЖТ `sess.Events` closes тЖТ `Listen` goroutine
exits.

and the `/sessions` slash command when the journal (Chunk 6) and SQLite checkpoint
