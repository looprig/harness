# Session Layer Design

**Date:** 2026-06-10
**Branch:** feat/agent-loop

## Problem

`internal/agent/session` is a 26-line file that generates a UUID and wraps `loop.Loop`. It is agent-specific and has no equivalent for graph-backed workflows. As we add graph workflows, both execution modes need the same runtime identity contract: a session ID, a checkpoint store binding, and a stable handle the client can use to send input and receive output.

## Goals

1. Unify runtime identity under `internal/session/` for both agent loops and graph workflows.
2. Introduce `internal/agent/agent.go` (Agent definition) and `internal/graph/workflow.go` (Workflow definition) as the configuration layer above the execution engines.
3. Introduce `internal/registry` — an in-process registry that maps names to agent/workflow definitions; provides `List` for discovery and `Lookup` for dispatch.
4. Rewire `client.Client[I, O]` to accept a name, look it up in the registry, and create the appropriate session. Both `cmd/cli` and `console` go through `client.New(ctx, r, name)`.
5. Both `cmd/cli` and `console` call gateway for AuthN/Z. CLI authenticates once at startup; console authenticates per inbound request. `client.New` is called on user selection (CLI) or per API request (console).
6. Delete `internal/agent/session/session.go` and update all call sites.

## Mental model

```
gateway                   — AuthN/Z (separate process)
                            CLI: authenticate once at startup
                            console: authenticate per inbound request
        ↓ allow/deny
cmd/cli                   console
  authenticate at startup   authenticate per request
  list registry             receive (token, name, input)
  user selects name         client.New per request
  client.New on selection
        ↓
client.Client[I, O]       — looks up name in registry; creates AgentSession or
                            WorkflowSession; owns its lifecycle
                            exposes Invoke/Stream directly
        ↓
internal/registry         — in-process *Registry with New()
                            List() []string for discovery
                            Lookup(name) for dispatch
        ↓
internal/session          — runtime identity: UUID, checkpoint store binding, engine handle
        ↓
loop.Loop / Runnable      — execution engines: turn runner / graph executor
```

### Import layering

```
internal/registry →  internal/agent            (agent.Agent)
                  →  internal/graph             (graph.Workflow)

internal/session  →  internal/agent            (agent.Agent)
                  →  internal/agent/loop        (loop.Loop, loop.Config, loop.Event)
                  →  internal/graph             (graph.Workflow, graph.Runnable, graph.Option)
                  →  internal/interrupt         (CheckPointStore)
                  →  internal/hooks             (SetExecutionInfo, ExecutionInfo)
                  →  internal/stream            (Pipe, StreamReader)

client            →  internal/registry
                  →  internal/session
                  →  internal/graph             (graph.Workflow — for type assertion in New[I,O])
                  →  internal/agent/loop        (loop.Event — for Client[string, loop.Event])
```

`internal/registry` does not import `internal/session` — dispatch to `session.NewAgent` / `session.NewWorkflow` is the client's responsibility, not the registry's. None of `internal/agent`, `internal/agent/loop`, `internal/graph`, or `internal/interrupt` may import `internal/session` or `internal/registry`. Violations break the layering rules in CLAUDE.md.

### Execution state vs domain data vs checkpoint

| | Execution state | Domain data | Checkpoint |
|---|---|---|---|
| **Graph** | channel states, node inputs, step position | typed `I`/`O` per run | snapshot of execution state, keyed by `checkPointID` |
| **Agent loop** | `loopState{turnIndex, status}` | `[]*AgenticMessage` (conversation history) | not yet implemented — session is the foundation |

A checkpoint is a serialized snapshot of execution state written to a `CheckPointStore` keyed by session ID. The graph has full checkpoint support today. The agent loop does not yet — when added, session owns both the ID (the key) and the store (where to write).

When loop checkpointing is implemented, the stored type must be named `checkpoint` (same as the graph's unexported `checkpoint` struct, within `internal/agent/loop`) — not `loopCheckpoint` or `agentState`. The live in-memory form stays `loopState`. This keeps the live/stored naming symmetric across both execution engines.

## New types

### `internal/agent/agent.go`

Package: `package agent` (sibling of `internal/agent/loop/` — Go allows a directory with its own package and subdirectories with different packages).

```go
package agent

import (
    "github.com/cloudwego/eino/internal/agent/loop"
    "github.com/cloudwego/eino/internal/interrupt"
)

// Agent is a pure definition — no runtime state.
// It provides everything needed to construct an agent session.
type Agent struct {
    Config loop.Config
    Store  interrupt.CheckPointStore // nil = no persistence
}
```

### `internal/agent/loop` — new `TurnInterrupted` event

`loop.Event` gains a new variant:

```go
// TurnInterrupted is emitted when Interrupt() is called during a running turn.
// Accumulated tokens up to the interrupt point are discarded.
// This is a first-class outcome, not an error.
type TurnInterrupted struct{}
```

Both `AgentSession.Invoke` and `AgentSession.Stream` treat `TurnInterrupted` as a normal event — not a Go error. The loop emits it and stops the current turn.

### `internal/graph/workflow.go`

```go
package graph

import "github.com/cloudwego/eino/internal/interrupt"

// Workflow is a pure definition — no runtime state.
// It provides everything needed to construct a workflow session.
type Workflow[I, O any] struct {
    Graph *Graph[I, O]
    Store  interrupt.CheckPointStore // nil = no persistence
}
```

Both `agent.Agent` and `graph.Workflow` use `interrupt.CheckPointStore` directly — same type, same contract.

### `internal/session/agent.go`

```go
package session

import (
    "context"
    "errors"

    "github.com/cloudwego/eino/internal/agent"
    "github.com/cloudwego/eino/internal/agent/loop"
    "github.com/cloudwego/eino/internal/interrupt"
    "github.com/cloudwego/eino/internal/stream"
)

// ErrConcurrentCall is returned when Invoke or Stream is called while
// another call is already in progress on the same AgentSession.
var ErrConcurrentCall = errors.New("session: concurrent Invoke/Stream not allowed")

// AgentSession is a running agent loop with a stable identity.
type AgentSession struct {
    SessionID string
    loop   *loop.Loop
    store  interrupt.CheckPointStore // retained for loop checkpointing follow-on
    active int32                     // atomic: 1 if Invoke/Stream in progress
}

// NewAgent generates a session ID and starts the agent loop.
func NewAgent(ctx context.Context, a agent.Agent) (*AgentSession, error)

// Invoke sends text into the loop and blocks until TurnDone, TurnFailed,
// or TurnInterrupted. Returns the terminal event or a Go error on failure.
// Returns ErrConcurrentCall if another Invoke/Stream is in progress.
func (s *AgentSession) Invoke(ctx context.Context, input string) (loop.Event, error)

// Stream sends text into the loop and returns a StreamReader[loop.Event]
// that yields events (TurnStarted, TokenDelta×N, then TurnDone/TurnFailed/TurnInterrupted).
// The StreamReader is closed after the terminal event.
// Returns ErrConcurrentCall if another Invoke/Stream is in progress.
func (s *AgentSession) Stream(ctx context.Context, input string) (*stream.StreamReader[loop.Event], error)

// Interrupt signals the loop to stop the current turn.
// The running Invoke/Stream receives TurnInterrupted as a first-class event — not an error.
func (s *AgentSession) Interrupt()

// Shutdown sends a shutdown command and waits for the loop goroutine to exit.
func (s *AgentSession) Shutdown()
```

**`Invoke` internals:** sends a `turnCommand` into `Commands`, then reads `Events` accumulating until it receives a terminal event (`TurnDone`, `TurnFailed`, or `TurnInterrupted`). Non-terminal events (`TurnStarted`, `TokenDelta`) are discarded. Returns the terminal event.

**`Stream` internals:** sends a `turnCommand` into `Commands`, creates a `stream.Pipe[loop.Event]`, spawns a goroutine that reads `Events` and writes each event to the pipe writer. Closes the writer on receipt of the terminal event. Returns the reader immediately.

**`Interrupt` internals:** sends a `controlCommand{Interrupt}` into `Commands`. The loop emits `TurnInterrupted` on the `Events` channel. The running `Invoke`/`Stream` goroutine receives it as a normal event — `Invoke` returns it, `Stream` writes it and closes.

**Concurrency:** `active int32` is set atomically at the start of `Invoke`/`Stream` and cleared on return. A second concurrent call returns `ErrConcurrentCall` immediately.

`NewAgent` generates a UUID, passes it to `loop.New`, and stores `a.Store` on the session. `NewAgent` is a direct replacement for `internal/agent/session.New`.

### `internal/session/workflow.go`

```go
package session

import (
    "context"
    "sync"

    "github.com/cloudwego/eino/internal/graph"
    "github.com/cloudwego/eino/internal/hooks"
    "github.com/cloudwego/eino/internal/stream"
)

// WorkflowSession wraps a compiled Runnable with a stable identity.
// Every Invoke/Stream call automatically uses the session ID as the checkpoint ID
// and injects it as GraphRunID in ExecutionInfo so all hook callbacks carry a
// stable correlation key across the session lifetime.
type WorkflowSession[I, O any] struct {
    SessionID   string
    runnable    graph.Runnable[I, O]
    mu          sync.Mutex
    interruptFn func(...graph.InterruptOption) // non-nil only while Invoke/Stream is in flight
    active      int32                          // atomic: 1 if Invoke/Stream in progress
}

// NewWorkflow compiles the graph with the checkpoint store baked in, generates a session ID.
// Returns an error if compilation fails.
func NewWorkflow[I, O any](ctx context.Context, w graph.Workflow[I, O]) (*WorkflowSession[I, O], error)

// Invoke runs the graph to completion and returns the typed output.
// Returns ErrConcurrentCall if another Invoke/Stream is in progress.
func (s *WorkflowSession[I, O]) Invoke(ctx context.Context, input I) (O, error)

// Stream runs the graph in streaming mode and returns a StreamReader[O].
// Returns ErrConcurrentCall if another Invoke/Stream is in progress.
func (s *WorkflowSession[I, O]) Stream(ctx context.Context, input I) (*stream.StreamReader[O], error)

// Interrupt signals the in-flight Invoke/Stream to stop at the next safe boundary.
// The graph checkpoints its state before stopping, so the next call resumes from that point.
// No-op if no call is in progress.
func (s *WorkflowSession[I, O]) Interrupt()

// Shutdown cancels any in-flight call and releases resources.
func (s *WorkflowSession[I, O]) Shutdown()
```

`NewWorkflow` calls `w.Graph.Compile(graph.WithCheckPointStore(w.Store))`, baking the store into the compiled `Runnable` as a `CompileOption`. Compilation can fail (invalid graph topology, type mismatches).

**`Invoke` and `Stream` internals:** both call `graph.WithInterrupt(ctx)` at the start of the call, which returns an interrupt-aware context and a `func(...graph.InterruptOption)`. They store that func in `s.interruptFn` (under `s.mu`) before calling the runnable, and nil it out on return (also under `s.mu`).

**`Interrupt` internals:** locks `s.mu`, reads `s.interruptFn`, unlocks, then calls the func if non-nil. The graph's task manager receives the signal via its cancel channel, writes a checkpoint, and returns an `interruptError`. The in-flight `Invoke`/`Stream` surfaces this as an error to its caller — unlike `AgentSession` where `TurnInterrupted` is a first-class non-error event, workflow interrupts are returned as errors (consistent with how the graph engine already works).

Both `Invoke` and `Stream`:
1. Prepend `graph.WithCheckPointID(s.SessionID)` to opts.
2. Inject `hooks.SetExecutionInfo(ctx, &hooks.ExecutionInfo{GraphRunID: s.SessionID})` before calling the underlying Runnable. The graph engine inherits `GraphRunID` from the incoming context (`graph_run.go:112-114`), so every hook callback carries `GraphRunID = SessionID` as the stable correlation key.

## `internal/registry`

An in-process registry mapping names to definitions. Constructed once at binary startup and passed into `client.New`. No global state — satisfies CLAUDE.md "no global state" rule.

```go
package registry

// Kind identifies whether an entry is an agent or workflow.
type Kind int

const (
    KindAgent    Kind = iota
    KindWorkflow
)

// Entry is returned by Lookup.
// def holds agent.Agent or graph.Workflow[I,O] (stored as any).
// client.New is responsible for type-asserting def and constructing the session.
type Entry struct {
    Kind Kind
    def  any
}

// Registry is an in-process name → definition store.
type Registry struct{ /* unexported map + mu */ }

func New() *Registry

// Register registers an agent definition under a name.
func (r *Registry) Register(name string, a agent.Agent)

// RegisterWorkflow registers a workflow definition under a name.
// Package-level generic function (Go does not allow generic methods).
// Stores w as any; client.New[I,O] type-asserts it back to graph.Workflow[I,O]
// and calls session.NewWorkflow — registry never imports session.
func RegisterWorkflow[I, O any](r *Registry, name string, w graph.Workflow[I, O])

// List returns all registered names sorted alphabetically.
func (r *Registry) List() []string

// Lookup returns the Entry for a name, or ErrNotFound if not registered.
func (r *Registry) Lookup(name string) (Entry, error)

var ErrNotFound = errors.New("registry: name not found")
```

`client.New` reads `entry.Kind` and branches:
- `KindAgent` → `session.NewAgent(ctx, entry.def.(agent.Agent))`
- `KindWorkflow` → `session.NewWorkflow[I,O](ctx, entry.def.(graph.Workflow[I,O]))`

## `client.Client` wiring

```go
// New looks up name in r and creates the appropriate session.
func New[I, O any](ctx context.Context, r *registry.Registry, name string) (*Client[I, O], error)

// Client[I, O] is fully typed over input and output.
// For agent: Client[string, loop.Event]
// For workflow: Client[MyInput, MyOutput]
type Client[I, O any] struct {
    sess Session[I, O]
}

// Session[I, O] is the shared interface over AgentSession and WorkflowSession.
type Session[I, O any] interface {
    Invoke(ctx context.Context, input I) (O, error)
    Stream(ctx context.Context, input I) (*stream.StreamReader[O], error)
    Interrupt()
    Shutdown()
}

// Invoke and Stream delegate directly to the underlying session.
func (c *Client[I, O]) Invoke(ctx context.Context, input I) (O, error)
func (c *Client[I, O]) Stream(ctx context.Context, input I) (*stream.StreamReader[O], error)
func (c *Client[I, O]) Interrupt()
func (c *Client[I, O]) Shutdown()
```

`AgentSession` satisfies `Session[string, loop.Event]`.
`WorkflowSession[I,O]` satisfies `Session[I, O]`.

No `Listen`, no `notify func`, no `Event` wrapper type — callers own their drain loops:
- **TUI** always calls `client.Stream(ctx, text)`, spawns a goroutine draining `StreamReader[loop.Event]`, calls `prog.Send(...)` on each event
- **console + agent** calls `client.Invoke(ctx, text)`, blocks, gets terminal `loop.Event`
- **console + workflow** calls `client.Invoke(ctx, input)`, blocks, gets typed `O`

The existing `client.New(commands chan<- Command)` is removed.

## Execution flows

### cmd/cli + Agent

1. CLI starts → gateway AuthN/Z once (interactive login)
2. `r.List()` → CLI shows available agents/workflows
3. User selects "cli-agent"
4. `client.New[string, loop.Event](ctx, r, "cli-agent")` → registry lookup → `session.NewAgent` → starts loop goroutine
5. TUI starts
6. User types → TUI calls `c.Stream(ctx, text)` → returns `StreamReader[loop.Event]` immediately
7. TUI goroutine drains `StreamReader`, calls `prog.Send(...)` on each event → renders TokenDelta, TurnDone in real time
8. User types again → repeat from 6
9. User quits → `c.Shutdown()`

### cmd/cli + Workflow

1. CLI starts → gateway AuthN/Z once
2. `r.List()` → CLI shows available agents/workflows
3. User selects "my-workflow"
4. `client.New[ResumeRequest, ResumeAnalysis](ctx, r, "my-workflow")` → registry lookup → `session.NewWorkflow` → compiles graph
5. TUI starts
6. User provides input → TUI calls `c.Stream(ctx, input)` → returns `StreamReader[ResumeAnalysis]` immediately
7. TUI goroutine drains `StreamReader`, `prog.Send(...)` on result → renders output
8. Done (workflow sessions resume from same SessionID checkpoint on next input)

### console + Agent

1. Inbound request arrives (token, agent-name, initial-input)
2. Gateway AuthN/Z
3. `client.New[string, loop.Event](ctx, r, "my-agent")` → registry lookup → `session.NewAgent` → starts loop goroutine
4. `c.Invoke(ctx, initial-input)` → blocks until terminal `loop.Event`
5. Agent runs autonomously, uses AskUser tool if needed, completes
6. `TurnDone` event returned → response sent to caller
7. `c.Shutdown()`

### console + Workflow

1. Inbound request arrives (token, workflow-name, input)
2. Gateway AuthN/Z
3. `client.New[MyInput, MyOutput](ctx, r, "my-workflow")` → registry lookup → `session.NewWorkflow` → compiles graph
4. `c.Invoke(ctx, input)` → blocks until graph completes
5. Typed `MyOutput` returned → response sent to caller
6. Done

## `cmd/cli/main.go` after rewiring

```go
import (
    "github.com/cloudwego/eino/client"
    "github.com/cloudwego/eino/internal/agent"
    "github.com/cloudwego/eino/internal/agent/loop"
    "github.com/cloudwego/eino/internal/registry"
)

// At startup: build registry and register definitions.
r := registry.New()
r.Register("cli-agent", agent.Agent{
    Config: loop.Config{GatewayURL: gatewayURL, Token: tokenFn, Model: ...},
})

// Authenticate with gateway once.
// gateway.Authenticate(ctx, ...) — out of scope, follow-on

// On user selection:
c, err := client.New[string, loop.Event](ctx, r, "cli-agent")
if err != nil {
    return fmt.Errorf("client: %w", err)
}
screen := tui.New(c)
prog := tea.NewProgram(screen, tea.WithAltScreen())

// TUI owns the drain loop — no Listen on client.
// tui.Model calls c.Stream(ctx, text) on user input and drains in a goroutine.
prog.Run()
```

`sess` and `Listen` disappear from `cmd/cli/main.go`. The client owns session lifecycle; the TUI owns event draining.

## Out of scope

- Agent loop checkpoint serialization (session is the foundation; implementation is a follow-on).
- Tool registration on `Agent` — added the day a real call site needs it.
- `console` binary — follow-on once the console entry point exists.
- Gateway AuthN/Z client implementation — follow-on; call site stubbed in `cmd/cli/main.go`.
- Dynamic registry (definitions outside the binary) — follow-on; `internal/registry` is in-process only for now.
