# Coding Agent Design

## Scope

Four connected pieces:

1. **`agents/coding`** — self-contained coding agent definition (system prompt + model sealed in)
2. **`client` catalog** — `client.Register` maps agent names to constructors; `client.New` creates a session from the populated registry; entrypoints never import agent packages directly
3. **Gateway catalog API** — `GET /v1/catalog` returns the list of agents the authenticated user can access; returns `{"agents": [{"name": "coding", ...}]}` for all authenticated users for now
4. **`cmd/cli` picker** — fetches catalog, auto-selects if one agent, shows picker if many, calls `client.New` with the chosen name

## 1. agents/coding

### Folder layout

```
agents/
└── coding/
    ├── agent.go          — New(gatewayURL, token) agent.Agent
    ├── prompts/
    │   └── system.go     — exported const SystemPrompt (package prompts)
    ├── evals/            — empty placeholder for evaluation harness
    └── golden-set/       — empty placeholder for golden I/O pairs
```

`evals/` and `golden-set/` are directories with `.gitkeep` files. No Go source yet.

### API

```go
package coding

// New returns a coding agent with the system prompt and model sealed in.
// gatewayURL is the base URL of the Nexus gateway.
// token is called before each request to obtain the current Bearer token.
func New(gatewayURL string, token func() (string, error)) agent.Agent
```

### System prompt design

A Go `const` — never constructed at runtime, never interpolated. Verbatim text
that lives in `prompts/system.go`.

## 2. client catalog

Registration and session creation are two separate steps. `internal/registry` stays
internal — nothing outside `client/` imports it or any agent package.

### Registry type

`client` exposes an opaque `Registry` that wraps `internal/registry.Registry`:

```go
// Registry is an opaque handle. Construct with NewRegistry, populate with Register.
type Registry struct{ r *registry.Registry }

func NewRegistry() *Registry
```

### Registering agents

```go
// Register looks up name in the internal agent catalog, constructs the agent definition,
// and stores it in reg. Returns an error if name is not in the catalog.
func Register(reg *Registry, name, gatewayURL string, token func() (string, error)) error
```

Internal catalog (unexported):

```go
// catalog.go
var catalog = map[string]func(string, func() (string, error)) agent.Agent{
    "coding": coding.New,
}
```

Adding a new agent type = one line in the catalog. No changes to `cmd/cli`.

### Input type change — adapter in client/

`AgentSession` today implements `Session[string, loop.Event]` — `Invoke` takes a
plain `string`. That needs to become `[]*content.Block` so it can carry images and
files. `internal/session` cannot import `client` (layering: `client` imports
`internal/session`, not the reverse), so the change is two steps:

**Step 1 — `internal/session`: change `AgentSession` from `Session[string, ...]` to `Session[[]*content.Block, ...]`**

`AgentSession.Invoke` and `Stream` change their input parameter from `string` to
`[]*content.Block`. The loop's `StartTurn` already takes `[]*content.Block`, so
the body becomes a direct pass-through. Delete the `inputBlocks` helper at the
bottom of `session/agent.go` — it wrapped a string into a block and is no longer needed.

**Step 2 — `client/`: adapter bridges `client.Input` → `[]*content.Block`**

`client/` introduces an unexported adapter that satisfies `Session[*Input, Event]`:

```go
// client/client.go
type agentSessionAdapter struct{ s *session.AgentSession }

func (a *agentSessionAdapter) Invoke(ctx context.Context, input *Input) (Event, error) {
    return a.s.Invoke(ctx, input.Blocks())
}
func (a *agentSessionAdapter) Stream(ctx context.Context, input *Input) (*stream.StreamReader[Event], error) {
    return a.s.Stream(ctx, input.Blocks())
}
func (a *agentSessionAdapter) Interrupt() { a.s.Interrupt() }
func (a *agentSessionAdapter) Shutdown()  { a.s.Shutdown() }
```

`Event` is a type alias exported from `client` so entrypoints never import `loop`:

```go
// client/event.go
type Event = loop.Event
```

The compile-time assertions in `client/client.go` change from:
```go
var _ Session[string, loop.Event] = (*session.AgentSession)(nil)
```
to:
```go
var _ Session[[]*content.Block, loop.Event] = (*session.AgentSession)(nil)
var _ Session[*Input, Event]                = (*agentSessionAdapter)(nil)
```

`WorkflowSession[I,O]` is unaffected — its type parameters come from the compiled
graph and are independent.

### Creating a session

`New[I,O]` signature is unchanged. Two changes to the implementation:
1. Registry parameter: `*registry.Registry` → `*client.Registry` (reads `reg.r` internally)
2. The `KindAgent` branch changes: `AgentSession` now satisfies `Session[[]*content.Block, loop.Event]`, not `Session[*Input, Event]`. The branch must wrap the session in `agentSessionAdapter` instead of doing a bare type assertion:

```go
case registry.KindAgent:
    a := entry.Def().(agent.Agent)
    as, err := session.NewAgent(ctx, a)
    if err != nil {
        return nil, err
    }
    typedSess, ok := any(&agentSessionAdapter{s: as}).(Session[I, O])
    if !ok {
        return nil, errors.New("client: agent session requires I=*Input and O=loop.Event")
    }
    sess = typedSess
```

The `KindWorkflow` branch is unchanged.

```go
func New[I, O any](ctx context.Context, reg *Registry, name string) (*Client[I, O], error)
```

### cmd/cli usage

```go
reg := client.NewRegistry()
for _, a := range catalogAgents {
    if err := client.Register(reg, a.Name, gatewayURL, tokenFn); err != nil {
        return fmt.Errorf("register agent %q: %w", a.Name, err)
    }
}
c, err := client.New[*client.Input, client.Event](ctx, reg, name)
```

`cmd/cli` imports only `client`. No `internal/registry`, no `loop`, no agent packages.

### Error behavior

`Register` returns an error if the name is not in the internal catalog.
`New` returns an error if the name was never registered.

## 3. Gateway catalog API

### Endpoint

```
GET /v1/catalog
Authorization: Bearer <token>
```

Auth middleware runs first — unauthenticated requests get 401 before reaching this handler.

### Response

```json
{
  "agents": [
    { "name": "coding", "description": "Expert coding assistant. Reads and edits files, runs tests, debugs and refactors code." }
  ]
}
```

### Authorization logic (v1)

Return `{"agents": [{"name": "coding", ...}]}` for all authenticated users regardless
of identity. No per-user authorization checks. When per-user AuthZ lands, this is the
only place that changes.

### Handler location

`gateway/adapter/http/catalog.go` adds a `handleCatalog` method on `Handler`.
Registration happens inside `NewHandler` in `gateway/adapter/http/handler.go`,
alongside the existing health endpoint:

```go
if auth != nil {
    h.mux.Handle("GET /v1/catalog", auth.Wrap(http.HandlerFunc(h.handleCatalog)))
} else {
    h.mux.Handle("GET /v1/catalog", http.HandlerFunc(func(w http.ResponseWriter, _ *http.Request) {
        w.WriteHeader(http.StatusServiceUnavailable)
    }))
}
h.mux.Handle("GET /v1/health", http.HandlerFunc(h.handleHealth)) // existing, unauthenticated
h.mux.Handle("/", proxied)                                        // existing catch-all
```

When `auth` is nil (`NEXUS_NO_AUTH=true`), the catalog handler returns 503 Service
Unavailable — it is not available without authentication. The `auth.Wrap` call must
be guarded; calling it on a nil `authWrapper` panics.

## 4. cmd/cli picker flow

### Flow

1. After auth, fetch `GET /v1/catalog` with the user's Bearer token; build `reg` via `client.NewRegistry()` + `client.Register()` for each returned agent
2. If fetch fails — fatal error, exit
3. If one agent returned — auto-select, skip picker
4. If multiple agents returned — render a list in the TUI, user picks one
5. Pass chosen name to `client.New[*client.Input, client.Event](ctx, reg, name)`

In no-auth mode (`NEXUS_NO_AUTH=true`), no catalog fetch is made and no agents are registered — the CLI exits with a clear message that no-auth mode does not support agents.

`cmd/cli` must not import `internal/registry` or any agent package; all registry
interaction goes through the `client` API.

### Picker UI

A simple numbered list rendered before the TUI chat screen opens. Not a bubbletea
component — a plain `fmt.Fprintf` + `bufio.Scanner` prompt is enough for v1:

```
Available agents:
  1. coding — Expert coding assistant
  2. data-analyst — ...

Select agent [1]: _
```

The picker only appears when 2+ agents are returned. For v1 it never appears (only
"coding" is returned), but the plumbing must be in place.

## 5. TUI client type change

`client/cli/tui/model.go` today holds `*client.Client[string, loop.Event]` and
calls `m.client.Stream(ctx, text)` with a raw string. After the `AgentSession`
input type change, this must become `*client.Client[*client.Input, client.Event]`.

Three changes in `model.go`:

1. `Screen.client` field: `*client.Client[string, loop.Event]` → `*client.Client[*client.Input, client.Event]`
2. `tui.New` parameter: same type change
3. `drain`: wrap textarea text in `client.Input` before calling `Stream`:
```go
func (m Screen) drain(ctx context.Context, text string) {
    sr, err := m.client.Stream(ctx, client.NewInput(text))
    ...
}
```

The `eventMsg` type stays as `loop.Event` internally — the TUI already imports
`loop` directly for the event type switch, and that is acceptable since `tui` is
not a binary entrypoint. Only `cmd/cli` must not import internal packages.

client/               — typed Client[I,O]; imports internal/registry, internal/session,

diagram between `internal/agent/` and `client/`.

## 6. Files that must be updated (not created)

### `client/client_test.go`

After the `*client.Registry` opaque wrapper and `*Input` type parameter changes, the
existing test uses `registry.New()` directly and calls `client.New[string, loop.Event]`.
Both break. The test must be rewritten to use `client.NewRegistry()` +
`client.Register()` and `client.New[*client.Input, client.Event]`.

### `gateway/adapter/http/handler_test.go`

Adding `GET /v1/catalog` requires new test cases alongside the existing ones:
- `auth=nil` → `GET /v1/catalog` returns 503 (no panic)
- `auth` set → `GET /v1/catalog` returns 200 with the expected JSON body

Both cases follow the existing table-free, inline-server pattern already used in `handler_test.go`.

### `internal/session/agent_test.go`

After `AgentSession.Invoke` and `Stream` change from `string` to `[]*content.Block`,
the existing tests only call `NewAgent` and check session IDs — they still compile.
But a coverage gap opens: add a test case that calls `Invoke` (or confirms it accepts
`[]*content.Block`) so the signature change is verified at the test level, not just
by the compile-time assertion in `client/client.go`.

### `cmd/cli/main.go` — full rework

Today `cmd/cli/main.go` directly imports and uses four internal packages:
`internal/agent`, `internal/agent/loop`, `internal/llm`, `internal/registry`. After
this spec all four must be removed. The `-model` and `-system` flags also become dead
code — the model and system prompt are sealed into `agents/coding`. The full new
shape:

```go
// 1. Auth (unchanged)
// 2. Fetch GET /v1/catalog with tokenFn
reg := client.NewRegistry()
for _, a := range catalogAgents {
    if err := client.Register(reg, a.Name, gatewayURL, tokenFn); err != nil {
        return fmt.Errorf("register agent %q: %w", a.Name, err)
    }
}
// 3. Picker (auto-select if one agent)
name := pick(catalogAgents)
// 4. Create client — no internal imports
c, err := client.New[*client.Input, client.Event](ctx, reg, name)
```

`cmd/cli` imports only `client` and `client/cli/tui` (plus stdlib + bubbletea for
the TUI). No `internal/*`, no agent packages.

## What is explicitly out of scope

- Per-user authorization in the catalog — all authenticated users get the same list
- Persistence (`Store`) — nil for now; no checkpoint support
- Cache control flags (`CacheSystem`, `CacheTools`) — out of scope
- Per-language sub-prompts — add as a separate agent if needed
- Composability — system prompt cannot be extended or overridden by callers
