# LLM Package Updates — Design

**Date:** 2026-06-07
**Branch:** feat/agent-loop

---

## Motivation

The agent loop design requires two things the current `internal/llm/` package does not
support:

1. A clean way to configure reasoning/thinking per model family without provider-specific
   fields leaking into shared code.
2. A request shape that carries model identity, sampling params, and system prompt together
   as a unit — making primary/fallback symmetric and keeping callers simple.

This spec also fixes three existing problems: the `CacheControl.TTL` field that is modeled
but never read; the `Chat` method name that conflicts with agent framework conventions; and
stream tests that live in the wrong package.

---

## 1. `ModelSpec` — model identity + all parameters

`ModelParams` and the bare `Model string` on `ChatRequest` are replaced by a single
`ModelSpec` struct. Everything that configures *how a model behaves* lives here.

```go
// ModelSpec identifies a model and its sampling/reasoning configuration.
// Zero value is valid: Model="" means "not set", all numeric fields are provider defaults.
type ModelSpec struct {
    Model  string
    System string // system prompt; Anthropic encodes as top-level system block, OpenAI as first message

    Temperature     *float64
    TopP            *float64
    MaxTokens       *int
    Stop            []string

    // ThinkingBudget enables Anthropic extended thinking. Maps to thinking.budget_tokens.
    // When >0, Temperature must be exactly 1.0 — any other value (including nil) returns ValidationError.
    ThinkingBudget int

    // ReasoningEffort selects OpenAI o-series reasoning intensity.
    // Valid non-empty values: ReasoningEffortLow/Medium/High.
    // Empty = disabled. Silently ignored by Anthropic and MoonshotAI adapters.
    ReasoningEffort ReasoningEffort
}

// ReasoningEffort selects OpenAI o-series reasoning intensity.
type ReasoningEffort string

const (
    ReasoningEffortLow    ReasoningEffort = "low"
    ReasoningEffortMedium ReasoningEffort = "medium"
    ReasoningEffortHigh   ReasoningEffort = "high"
)
```

### Design rationale

`System` belongs in `ModelSpec` because it is static model configuration — set once per
agent node, never mutated across turns. Putting it in `Messages` would force every caller
to prepend it correctly each turn and ensure it never duplicates. The gateway encoder
always pulls `Primary.System` and places it at the correct wire position.

The fallback model can legitimately have a different system prompt (e.g. different model
family with different prompting conventions), so `System` in `ModelSpec` gives that for free.

## 2. `Request` — replaces `ChatRequest`

```go
// Request is the provider-neutral inference request.
type Request struct {
    Primary  ModelSpec // required; Model must be non-empty
    Fallback ModelSpec // Model == "" means no fallback

    Messages       []*AgenticMessage
    Tools          []Tool
    CacheSystem    bool // apply cache_control breakpoint to system block (Anthropic only)
    CacheTools     bool // apply cache_control breakpoint after last tool (Anthropic only)
    Stream         bool
    ResponseFormat *ResponseFormat

    // Extra carries provider-specific request knobs not in ModelSpec.
    // Keys are wire field names; each adapter reads only the keys it understands.
    // Silently ignored keys: adapters never return errors for unknown keys.
    //
    // Anthropic: "top_k" (int)
    // OpenAI:    "seed" (int), "presence_penalty" (float64), "frequency_penalty" (float64),
    //            "logprobs" (bool), "logit_bias" (map[string]int), "service_tier" (string)
    Extra map[string]any
}
```

### Fallback semantics

The gateway checks `req.Fallback.Model != ""` after a retryable error (5xx, NetworkError).
On fallback it retries once with `req.Fallback` as the model spec — same messages, tools,
and Extra. Non-retryable errors (4xx, ValidationError) skip fallback immediately.

`FailoverGateway` remains for multi-gateway failover (different endpoints/credentials).
`Request.Fallback` is the single-request convenience form (same gateway, different model).

### `Options` and `With*` functions — deleted

`Options`, `WithTemperature`, `WithMaxTokens`, `WithTopP`, `WithStop`, `WithCacheTools`,
`WithCacheSystem`, `WithResponseFormat`, `WithTools`, `ApplyOptions` are all deleted.

`internal/llm/` is an internal package. Adding a field to `ModelSpec` is caught by the
compiler at every construction site. Functional options add ceremony with no benefit.
Go stdlib uses plain structs for configuration (`http.Server`, `tls.Config`, `net.Dialer`).

Call sites that previously used options now set struct fields directly:

```go
// before
req := llm.ChatRequest{Model: "claude-sonnet-4-6", ModelParams: &llm.ModelParams{MaxTokens: ptr(1000)}}

// after
req := llm.Request{Primary: llm.ModelSpec{Model: "claude-sonnet-4-6", MaxTokens: ptr(1000)}}
```

## 3. `LLMClient` — `Chat` renamed to `Invoke`

`Chat` is chat-SDK terminology. `Invoke` is the standard term in agent frameworks for a
single blocking call. Rename throughout.

**`internal/graph/llm_node.go`**

```go
type LLMClient interface {
    Invoke(ctx context.Context, req llm.Request) (*llm.AgenticMessage, error)
    Stream(ctx context.Context, req llm.Request) (*stream.StreamReader[*llm.AgenticMessage], error)
}

type LLMNodeConfig struct {
    Client      LLMClient
    Primary     llm.ModelSpec
    Fallback    llm.ModelSpec // Model == "" means no fallback
    CacheSystem bool          // copied into Request.CacheSystem by buildRequest
    CacheTools  bool          // copied into Request.CacheTools by buildRequest
}
```

`buildRequest` constructs an `llm.Request` directly from `LLMNodeConfig` fields — no
options merging. `CacheSystem`/`CacheTools` live on both `LLMNodeConfig` (static node
config) and `Request` (wire-level instruction read by the encoder); `buildRequest` copies
them across. The per-call `...llm.Option` variadic is removed; `AnyFunc` is instantiated
with the `WithoutOption` variants (`InvokeWOOpt`, `StreamWOOpt` etc.).

**`internal/llm/providers/gateway.go`**

```go
func (g *Gateway) Invoke(ctx context.Context, req llm.Request) (*llm.AgenticMessage, error)
func (g *Gateway) Stream(ctx context.Context, req llm.Request) (*stream.StreamReader[*llm.AgenticMessage], error)
```

**`internal/llm/providers/failover.go`**

```go
func (f *FailoverGateway) Invoke(ctx context.Context, req llm.Request) (*llm.AgenticMessage, error)
func (f *FailoverGateway) Stream(ctx context.Context, req llm.Request) (*stream.StreamReader[*llm.AgenticMessage], error)
```

**`internal/llm/providers/middleware.go`**

```go
type InvokeFunc func(ctx context.Context, req llm.Request) (*llm.AgenticMessage, error)
type Middleware func(next InvokeFunc) InvokeFunc
func Apply(f InvokeFunc, middlewares ...Middleware) InvokeFunc
```

`StreamFunc`, `StreamMiddleware`, `ApplyStream` signatures are unchanged except the
`llm.ChatRequest` parameter type becomes `llm.Request`.

## 4. Adapter changes

### Anthropic (`internal/llm/providers/anthropic/encode.go`)

**Reasoning:**
- `ThinkingBudget > 0` → emit `"thinking": {"type": "enabled", "budget_tokens": N}`
- If `ThinkingBudget > 0` and `Temperature` is nil or `!= 1.0` → return `ValidationError`
- `ReasoningEffort` silently ignored

**Cache breakpoints — fix for dead TTL field:**

`CacheControl.TTL` is currently modeled but ignored — the encoder always emits
`{"type": "ephemeral"}`. Fix: read TTL and emit the correct shape.

```go
// CacheControl marks a cache breakpoint for Anthropic prompt caching.
// TTL: "" = default (5 min ephemeral), "1h" = extended TTL (Opus/Haiku/Sonnet 4.5+ only).
type CacheControl struct {
    TTL string `json:"ttl,omitempty"`
}
```

Encoder logic (applied in all three cache positions: system block, tools block, message blocks):

```
TTL == ""   → {"type": "ephemeral"}
TTL == "1h" → {"type": "ephemeral", "ttl": 3600}
anything else → ValidationError
```

**Extra keys read:** `"top_k"` (int) → `"top_k"` on wire.

### OpenAI (`internal/llm/providers/openai/encode.go`)

**Reasoning:**
- `ReasoningEffort != ""` → emit `"reasoning_effort": value`
- Invalid value (not low/medium/high) → return `ValidationError`
- `ThinkingBudget` silently ignored

**Extra keys read:** `"seed"`, `"presence_penalty"`, `"frequency_penalty"`, `"logprobs"`,
`"logit_bias"`, `"service_tier"` — each passed through to the wire as-is.

### MoonshotAI (`internal/llm/providers/moonshotai/encode.go`)

`ThinkingBudget`, `ReasoningEffort`, `CacheControl` all silently ignored — no changes.

## 8. `ImageBlock` deduplication

`llm.ImageBlock` and `content.ImageBlock` have identical fields (`MediaType`, `URL`, `Data`).
`ContentBlock.Image` switches to `*content.ImageBlock` and `llm.ImageBlock` is deleted.
The `llm/types.go` consolidation imports `internal/content` for this type only — no other
`content` types move into `llm`.

## 9. Full impact inventory

Every call site that changes as a result of this spec, beyond the primary files already described.

### `internal/llm/providers/gateway.go`
- `g.chat ChatFunc` field → `g.invoke InvokeFunc`
- `g.doChat` → `g.doInvoke`
- `encodeRequest(p provider, req llm.ChatRequest)` → `encodeRequest(p provider, req llm.Request)`
- `Gateway.Invoke` gains fallback logic: after retryable error, if `req.Fallback.Model != ""`, rebuild request with `Primary = req.Fallback` and call `g.invoke` once more. Non-retryable errors skip fallback immediately.

### `internal/llm/providers/middleware.go`
- `ChatFunc` → `InvokeFunc`
- `Middleware func(next ChatFunc) ChatFunc` → `Middleware func(next InvokeFunc) InvokeFunc`
- `Apply(f ChatFunc, ...)` → `Apply(f InvokeFunc, ...)`
- Logging middleware: `req.Model` → `req.Primary.Model`

### `internal/llm/providers/failover.go`
- `FailoverGateway.Chat` → `FailoverGateway.Invoke`
- Internal `gw.Chat(ctx, req)` calls → `gw.Invoke(ctx, req)`

### `internal/llm/providers/{anthropic,openai,moonshotai}/encode.go`
- `EncodeRequest(req llm.ChatRequest)` → `EncodeRequest(req llm.Request)`
- All `req.ModelParams.*` field reads → `req.Primary.*`
- `req.Model` → `req.Primary.Model`
- `req.System` → `req.Primary.System`

### `internal/graph/llm_node.go`
- `LLMClient.Chat` → `LLMClient.Invoke`
- All `cfg.Client.Chat(...)` → `cfg.Client.Invoke(...)`
- `buildRequest` drops `...llm.Option` parameter; reads `cfg.Primary`, `cfg.Fallback`, `cfg.CacheSystem`, `cfg.CacheTools` directly
- `AnyFunc` instantiated with `WithoutOption` variants (`InvokeWOOpt`, `StreamWOOpt`, `CollectWOOpt`, `TransformWOOpts`)

### Test files (mechanical updates, no new test logic)
- `internal/llm/providers/middleware_test.go`: `providers.ChatFunc` → `providers.InvokeFunc`; `llm.ChatRequest{}` → `llm.Request{}`
- `internal/llm/providers/failover_test.go`: `newTestGatewayWithFunc(ChatFunc)` → `newTestGatewayWithFunc(InvokeFunc)`; `fg.Chat(...)` → `fg.Invoke(...)`
- `internal/llm/providers/gateway_test.go`: `llm.ChatRequest{Model: "..."}` → `llm.Request{Primary: llm.ModelSpec{Model: "..."}}`; middleware test that reads model name reads `req.Primary.Model`
- `internal/llm/providers/{anthropic,openai,moonshotai}/encode_test.go`: `llm.ChatRequest{..., ModelParams: &llm.ModelParams{...}}` → `llm.Request{Primary: llm.ModelSpec{...}}`
- `internal/llm/providers/gateway_integration_test.go`: same `ChatRequest`/`ModelParams` → `Request`/`ModelSpec` updates; `gw.Chat(...)` → `gw.Invoke(...)`
- `internal/graph/llm_node_test.go`: `fakeLLMClient.Chat` → `fakeLLMClient.Invoke`; `LLMNodeConfig{Model: "..."}` → `LLMNodeConfig{Primary: llm.ModelSpec{Model: "..."}}`
- `internal/graph/graph_llm_integration_test.go`: same `LLMNodeConfig` and `gw.Chat`/`ChatRequest` updates
- `internal/llm/options_test.go`: **deleted** entirely (tests `Options`/`With*` which are removed)

## What is explicitly out of scope

- Streaming reasoning tokens — already handled by existing `ThinkingBlock` decode path
- Budget enforcement client-side — providers enforce their own limits
- `top_k` validation — passed through as-is; Anthropic will reject invalid values at the wire
- Extended TTL availability enforcement — caller's responsibility to only use `"1h"` on
  supported model versions
