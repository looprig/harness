# v0.9.0 - Codex foreign-loop backend

_Released 2026-07-10 · tag `v0.9.0` · [compare v0.8.0...v0.9.0](https://github.com/looprig/harness/compare/v0.8.0...v0.9.0)_

Codex CLI can now back a primary loop or subagent through the existing foreign-loop and session contracts, including durable resume across restore.

## Added
- **`pkg/foreignloop/codex`:** a subprocess adapter for `codex exec --json` and `codex exec resume`, with typed configuration, a whitelisted environment, supported approval-policy overrides, JSONL decoding, fuzz coverage, and opt-in CLI contract tests.
- **Late-bound sessions:** durable `ForeignSessionBound` events record the Codex thread id learned from `thread.started`; restore recovers that id and resumes the same foreign session.
- **Engine selection:** `loop.EngineForeignCodex` selects the Codex adapter while preserving existing native and Claude behavior.

## Fixed
- Late-bound loops use independent temporary locks and atomically transition to the durable `(foreign SID, cwd)` lock before publishing the bound session id.
- Process exits, decode failures, premature EOF, missing or out-of-order init events, and mixed terminal failures now produce `TurnFailed` while retaining typed causes.
- A failed Codex start cannot resume with an empty SID; the loop retries `StartNew` until a nonempty session id has been bound.

## Notes
- The full repository test suite passes with the race detector. Focused lock, failure-ordering, parser, and restore tests were repeated during review.
- The production argv parser check runs with isolated configuration and no model request. The credentialed live Codex start/resume contract remains opt-in through `LOOPRIG_CODEX_INTEGRATION=1`.
- `make secure` remains blocked by pre-existing repository-wide formatting drift; Go 1.26.4 also reports standard-library advisories fixed by Go 1.26.5.
