# v0.8.0 - Loop-targeted session input

_Released 2026-07-10 · tag `v0.8.0` · [compare v0.7.0...v0.8.0](https://github.com/looprig/harness/compare/v0.7.0...v0.8.0)_

Sessions can now submit user input to a specific loop instead of routing every submission through the primary loop.

## Added
- **`pkg/session`:** `Session.SubmitToLoop` validates the target loop, journals the command, and delivers it through the existing loop command path. Tests cover primary, subagent, unknown-loop, and closed-session behavior.
- **Planning docs:** revised the client web application design and added the `pkg/secret` initialization plan.

## Changed
- Archived the unused `pkg/transcript` implementation and removed its Goldmark dependency and vendored sources from harness.

## Notes
- This release is additive for session clients: existing primary-loop submission behavior is unchanged.
