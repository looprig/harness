# v0.2.1 — Animated status line (gradient label + dot, above the input)

_Released 2026-06-29 · tag `v0.2.1` · [compare v0.2.0…v0.2.1](https://github.com/looprig/harness/compare/v0.2.0...v0.2.1)_

The active-surface status line moves above the input box, and its text now animates as a flowing lime↔blue gradient — label and dot together — replacing the old lime/white blink pulse.

## Changed
- **Status-line position (`pkg/tui`):** the status row (`idle` / `waiting…` / `thinking…` / `streaming…`) now renders ABOVE the bottom box, set off by a blank pad line above and below it. The live tail's trailing blank doubles as the pad-above, so the gap never grows to two rows; the tail budget reserves the extra pad. The rotating Tips line stays pinned to the very bottom.
- **Animated gradient label (`pkg/tui`):** the status label is colored per glyph by a cosine wave swept between `#D5F84D` and `#A2D2FF`, flowing with the live animation frame and rendering a static gradient at rest. Colors are interpolated in RGB (stdlib only).
- **Gradient status dot (`pkg/tui`):** the status dot now rides the same band (sampled two columns left of label glyph 0) instead of pulsing lime↔white, so the dot and label read as one continuous flow. It is a hollow ring at rest and a filled dot while a turn is live; the Running sub-state is carried by the label, so the dot needs only the session status to pick its glyph. The band advances ~1.5 glyphs per blink tick.

## Notes
- TUI-only change; no public API, session, or journal behavior is affected. The blink machinery still drives the live-tail assistant dot — only the status dot stopped consulting it (the now-dead `surfaceInputs.Blink` was removed).
- Verified on the changed package: `go test -race ./pkg/tui/`, `go vet`, gofmt, and a full `go build ./...`.
