# v0.2.0 — Pluggable foreign-loop backend (Claude Code headless)

_Released 2026-06-26 · tag `v0.2.0` · [compare v0.1.2…v0.2.0](https://github.com/looprig/harness/compare/v0.1.2...v0.2.0)_

A looprig session — primary loop or any subagent — can now be backed by an external coding agent, driven through the same `loop.Backend` contract. Claude Code (`claude -p`, headless) is the first adapter; the foreign session id is the durable handle.

## Added
- **`loop` (additive only at v0.2):** an `Engine` selector on the then-current mutable loop configuration (zero value = native, no behavior change) and a narrow `loop.Backend` interface (`CommandSink`/`DoneChan`/`Snapshot`) that both the native and foreign loop actors satisfy. The mutable construction surface was later replaced by declarative `loop.Define` options.
- **`pkg/foreignloop`:** the foreign-loop actor — `TurnStarted` (before spawn) → live token/tool events → transcript-authoritative `StepDone` → `TurnDone`; spawn-failure → `TurnFailed`; transcript-loss soft-degrade; `Interrupt` → `TurnInterrupted`; a per-`(sid,cwd)` liveness lock (busy guard, fails closed on a live stale holder); and restore via `NewRestored`. Includes a stream-json decoder, a version-tolerant transcript decoder (soft-degrade), an event mapper with tool-call id correlation, and fuzz targets on both decoders.
- **`pkg/foreignloop/claude`:** the real subprocess adapter — argv builder, transcript-path derivation with a traversal guard, env whitelist (never `os.Environ()` wholesale), process-group `Spawn` with `--session-id`/`--resume`, and a turn-key `Spec` resolver for composition-root wiring.
- **`pkg/session` at v0.2:** the then-current foreign-builder option, an `Engine` switch at the single loop-construction chokepoint, `LoopStarted.ForeignSID` (journaled durable handle), restore-by-sid (fail-closed on an empty sid), and cwd/adapter/posture folded into the config fingerprint. Composition later moved to `rig.WithForeignBuilders`.

## Changed
- `pkg/session` now drives loops through `loop.Backend` instead of the then-public concrete native actor (mechanical refactor, no behavior change).

## Fixed
- Foreign `TurnStarted` now stamps `Cause.CommandID`, so the session's drain correlates a foreign turn — without it `RunSubagent` over a foreign loop would hang.

## Notes
- `pkg/loop` diff is additive only (Engine, Backend, two accessors, one `LoopStarted` field) — no native behavior changed.
- Quality gates green: full `go test -race ./...`, `staticcheck`/`gosec`/`govulncheck`, gofmt, and `-trimpath` build. The Claude integration test runs when `claude` + `ANTHROPIC_API_KEY` are present and skips cleanly otherwise.
- **Known limitation:** the foreign actor does not emit `LoopIdle`, so a foreign *primary* loop won't drive a session to `SessionIdle` quiescence (subagents are unaffected — they drain to `TurnDone`). CLI flag/gate wiring is deferred to the composition-root branch.
